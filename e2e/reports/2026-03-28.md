## Fixes applied this session

### J24 — Duplicate stamp at same shop
**Root causes (3):**
1. `img[src^="blob:"]` photo preview never appeared → React's `PhotoUploader` wasn't fully hydrated when `setInputFiles` fired. Fix: wait for `waitForLoadState('networkidle')` + "Take Photo" button visible before `setInputFiles`.
2. `[data-sonner-toast]` toast assertion after `router.back()` → toast fires on `/checkin` then router navigates away; toast is gone before assertion. Fix: replaced with `waitForResponse` on `/api/checkins` POST (same pattern as J10).
3. Stamp count capped at 3 by `PolaroidSection`'s `MAX_PREVIEW = 3` → test was checking `[data-testid="memory-card"]` count which always maxed at 3. Fix: use "X recent visits" total count text (`stamps.length`) + re-navigate on each `toPass` retry to flush DB trigger delay.
4. Bump `test.setTimeout(90_000)` — two full check-in flows + DB-trigger retry loop exceed the 30s global timeout under full-suite load.

### J26 — Delete list
**Root causes (3):**
1. Broken list card selector (`li, article, [data-list-id]`) matched nothing — actual DOM uses plain `div` cards. Fix: `page.locator('div').filter({ hasText }).filter({ has: 'List options' button })`.
2. `window.confirm()` native dialog auto-dismissed as "Cancel" (Playwright doesn't see it as a DOM button). Fix: `page.once('dialog', d => d.accept())` before clicking Delete.
3. J26 ran outside the serial block, racing with J12/J13 on list cap. Fix: moved into serial block.
4. Residual "Delete Test" lists from prior failed runs accumulate and cause J27 to hit cap. Fix: J26 pre-cleanup deletes ALL lists (not just from index 2+).

### J12/J13 — List cap tests
**Root cause:** J26 and J27 were outside the serial block and ran in parallel with J12/J13, creating lists and hitting cap. Fix: moved all 4 tests into `test.describe.serial`.

**Root cause (cross-project race):** Mobile and desktop share the same test account. Desktop running list tests concurrently with mobile causes unavoidable cap races. Fix: `test.beforeEach` skips all list tests when `testInfo.project.name !== 'mobile'`.

**Root cause (stale residue):** J12's pre-cleanup only removed lists from index 2+, leaving 2 stale lists that would push J27 to cap. Fix: pre-cleanup now deletes ALL lists for a clean slate.

### J25 — Display name update
**Root cause:** Both mobile and desktop overwrite display name concurrently; only one name survives; the other project's assertion fails. Fix: skip on desktop (same shared-account pattern as lists).

### J38 — Account deletion cancel
**Root cause:** Test takes ~22s in isolation; 30s global timeout too tight under full-suite load. Fix: `test.setTimeout(60_000)`.

### J40 — Follow/unfollow toggle (first test)
**Root cause:** J38 initiates account deletion concurrently; follow API returns 4xx while account is in pending-deletion state; optimistic follow update rolls back. Fix: added proactive cancel-deletion step at start of test body + wrapped click→assert in `toPass()` retry pattern (matching the existing second test).

### playwright.config.ts — Worker cap
**Root cause:** `workers: undefined` (all CPU cores) allows ~16 concurrent browser workers; local Next.js dev server drops connections (ECONNRESET, socket hang up) under this load. Fix: `workers: 2` locally, `workers: 1` in CI.
