# Design: PWA Tier 1 — Installable App

Date: 2026-03-16

## Goal

Make CafeRoam installable on iOS and Android home screens (Add to Home Screen). Tier 1 only — installability + meta tags. No service worker, no offline support (Tier 2 post-launch).

## Scope

- Web app manifest via Next.js native `app/manifest.ts`
- Placeholder PWA icons generated by a one-shot script
- Layout metadata: viewport theme-color, icons, appleWebApp
- No service worker, no push notifications, no install prompt interception

## Architecture

### File Layout

```
scripts/generate-pwa-icons.ts    # one-shot icon generator (devDep: @napi-rs/canvas)
app/manifest.ts                  # Next.js manifest route (typed MetadataRoute.Manifest)
app/layout.tsx                   # add viewport, icons, appleWebApp metadata
public/
  icon-192.png                   # standard PWA icon
  icon-512.png                   # standard PWA icon (splash screen)
  icon-512-maskable.png          # maskable icon for Android adaptive icons
  apple-touch-icon.png           # 180x180 for iOS
  favicon.ico                    # 32x32 favicon
```

### Icon Generation

- **Character:** 啡 (from 啡遊)
- **Background:** `#6F4E37` (coffee brown)
- **Text color:** `#FFFFFF`
- **Font:** bold sans-serif, ~70% of icon width
- **Maskable variant:** content within center 80% (Android safe zone)
- **Sizes:** 192x192, 512x512, 512x512 maskable, 180x180 (apple-touch), 32x32 (favicon)
- **Runtime:** `@napi-rs/canvas` (pure Node, no native deps), run via `npx tsx scripts/generate-pwa-icons.ts`
- **Output:** committed PNGs in `public/`. Script is a dev tool, not part of the build.

### Manifest (`app/manifest.ts`)

```typescript
import type { MetadataRoute } from 'next';

export default function manifest(): MetadataRoute.Manifest {
  return {
    name: '啡遊 CafeRoam',
    short_name: '啡遊',
    description: "Discover Taiwan's best independent coffee shops",
    start_url: '/',
    display: 'standalone',
    background_color: '#ffffff',
    theme_color: '#6F4E37',
    icons: [
      { src: '/icon-192.png', sizes: '192x192', type: 'image/png' },
      { src: '/icon-512.png', sizes: '512x512', type: 'image/png' },
      {
        src: '/icon-512-maskable.png',
        sizes: '512x512',
        type: 'image/png',
        purpose: 'maskable',
      },
    ],
  };
}
```

### Layout Metadata Changes (`app/layout.tsx`)

**Add to `metadata` export:**

- `icons`: favicon.ico, apple-touch-icon.png, icon-192, icon-512
- `appleWebApp`: `{ capable: true, title: '啡遊', statusBarStyle: 'default' }`

**Add `viewport` export:**

- `themeColor: '#6F4E37'`
- `width: 'device-width'`, `initialScale: 1`, `maximumScale: 1`

## Decisions

| Decision          | Choice                            | Rationale                                               |
| ----------------- | --------------------------------- | ------------------------------------------------------- |
| Manifest approach | Next.js native `app/manifest.ts`  | Type-safe, no extra dependencies, built-in to framework |
| Icon generation   | `@napi-rs/canvas` one-shot script | Pure Node, no native deps, good CJK text rendering      |
| Display mode      | `standalone`                      | Native app feel, no browser chrome                      |
| Service worker    | None (Tier 1)                     | Installability doesn't require SW; offline is Tier 2    |
| Install prompt    | Browser native                    | No custom install banner; let the browser handle it     |

## Explicitly Excluded

- Service worker / offline caching (Tier 2)
- Push notifications
- Custom install prompt UI
- `next-pwa` or similar libraries
